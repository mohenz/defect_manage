# DefectFlow 모바일 테스트 사용자 매뉴얼

## 목적
- 모바일웹 테스트 중 `신고하기` 버튼을 이용해 결함을 빠르게 등록하는 절차를 설명합니다.
- 실제 테스트 담당자가 어떤 순서로 입력하고 확인해야 하는지 사용자 관점에서 정리합니다.

## 접속 정보
- DefectFlow 관리 화면: `https://mohenz.github.io/defect_manage/`
- 모바일 테스트 베드: `https://mohenz.github.io/defect_manage/test_view/mobile_test_bed.html`

## 사용 대상
- 모바일웹 화면을 수동 테스트하는 테스터
- 캡처 없이 빠르게 결함을 등록해야 하는 현장 점검 또는 스모크 테스트 담당자

## 사용 전 준비
1. DefectFlow 계정이 있어야 합니다.
2. 테스트할 모바일 페이지 또는 모바일 테스트 베드를 엽니다.
3. 로그인하지 않은 상태에서 등록 화면을 열면 로그인 화면으로 먼저 이동합니다.
4. 로그인 후에는 방금 열었던 등록 화면으로 자동 복귀합니다.

## 기본 사용 절차
1. 모바일 페이지 우측 하단의 `신고하기` 버튼을 누릅니다.
2. DefectFlow의 `모바일 간편 결함 등록` 화면이 열리면 항목을 확인합니다.
3. 필수 항목을 입력하거나 수정합니다.
4. `결함 등록` 버튼을 눌러 저장합니다.
5. 저장 후 DefectFlow 결함 목록에서 등록 결과를 확인합니다.

## 입력 항목 설명

### 필수 입력
- `결함 제목`
  - 최소 5자 이상 입력해야 합니다.
  - 예: `장바구니 버튼이 보이지 않음`
- `결함 발생 화면`
  - `FO (MO)` 화면 경로만 선택할 수 있습니다.
  - 실제 문제가 발생한 모바일 화면에 맞는 항목을 선택합니다.
- `결함 내용 또는 재현 절차`
  - 사용자가 무엇을 했고 어떤 현상이 나왔는지 적습니다.
  - 예: `상품 상세 진입 후 바로 구매 버튼이 사라짐`
- `테스트 구분`
  - 예: `통합테스트`, `단위테스트`
- `심각도`
  - 영향도가 큰 장애인지 판단해 선택합니다.
- `우선순위`
  - 처리 시급도를 선택합니다.

### 자동 또는 숨김 처리 항목
- `등록자`
  - 로그인한 사용자 이름이 자동으로 들어갑니다.
- `현재 페이지 URL`
  - 테스트 중이던 모바일 페이지 URL이 자동 저장됩니다.
- `이미지`
  - 모바일 간편 등록에서는 캡처 이미지를 사용하지 않습니다.
- `환경 정보`
  - 브라우저 정보와 연동 태그가 자동 저장됩니다.

## 화면별 권장 입력 예시

### 상품 목록 화면
- 결함 제목: `상품 카드 가격이 0원으로 표시됨`
- 결함 발생 화면: `FO (MO) > 메인화면 > 슬라이딩배너, 중분류`
- 결함 내용 또는 재현 절차:
  - `메인화면 진입`
  - `특가 상품 영역 확인`
  - `특정 상품 가격이 0원으로 노출`

### 상세 화면
- 결함 제목: `상세 화면에서 구매 버튼 클릭 불가`
- 결함 발생 화면: 실제 상세 화면에 해당하는 `FO (MO)` 항목 선택
- 결함 내용 또는 재현 절차:
  - `상품 상세 진입`
  - `구매 버튼 터치`
  - `반응 없음 또는 비활성 상태 지속`

## 저장 후 동작
- 저장에 성공하면 등록 화면의 임시 데이터는 삭제됩니다.
- standalone/mobile 모드에서는 저장 후 현재 팝업 또는 탭이 닫히는 흐름으로 동작합니다.
- 이후 DefectFlow 목록에서 방금 등록한 항목을 조회할 수 있습니다.

## 로그인 관련 동작
- 로그인하지 않은 상태에서 등록 화면을 열면 로그인 화면으로 먼저 이동합니다.
- 로그인 후에는 다시 등록 화면으로 복귀해 입력을 이어갈 수 있습니다.
- 같은 origin에서 연동된 페이지라면 위젯이 넣어 둔 임시 데이터가 유지됩니다.

## 점검 포인트
- 모바일 화면 경로를 정확히 선택했는지 확인합니다.
- 제목을 너무 짧게 쓰지 않았는지 확인합니다.
- 재현 절차가 다른 사람도 따라 할 수 있을 만큼 구체적인지 확인합니다.
- 저장 후 목록에서 등록자, 테스트 구분, 심각도 값이 기대한 대로 들어갔는지 확인합니다.

## 자주 발생하는 문제

### `신고하기` 버튼이 보이지 않음
- 페이지에 모바일 위젯이 연동되지 않았을 수 있습니다.
- 테스트 담당자에게 연동 여부를 먼저 확인합니다.

### 등록 화면은 열렸지만 값이 비어 있음
- 현재 페이지와 DefectFlow 등록 화면이 같은 origin이 아닐 수 있습니다.
- 이 경우 담당 개발자에게 연동 가이드 기준으로 same-origin 여부를 확인 요청합니다.

### 저장 버튼을 눌렀는데 등록되지 않음
- 제목이 5자 미만인지 확인합니다.
- 필수 입력값이 비어 있지 않은지 확인합니다.
- 로그인 세션이 만료되지 않았는지 확인합니다.

## 권장 운영 방식
- 모바일 스모크 테스트에서는 캡처보다 빠른 기록을 우선할 때 이 모드를 사용합니다.
- 상세한 이미지 근거가 꼭 필요하면 데스크톱 위젯 또는 Chrome 확장 흐름을 사용합니다.
- 테스트 종료 후에는 결함 목록에서 방금 등록한 항목을 한 번 더 열어 내용 누락 여부를 확인합니다.
