## [2026-03-30] Extension 설치 안내 화면 추가

### ✨ 기능 개선

#### 67. 결함 저장 오류 로그에 등록자명 저장 보강
- **배경**: 중앙 로그에서 실패 당시 결함의 등록자명을 바로 식별하기 어려워 원인 분석에 추가 확인이 필요했음
- **변경 사항**:
  - `defect_save_error_logs.reported_by`에 결함 등록자명(`payload creator`)을 저장하도록 조정
  - 실제 저장 시점 작업자명은 `extra.reporter_name`에 함께 남기도록 보강
  - 관리자 화면의 `저장 오류 로그` 목록과 상세 화면에서 등록자명을 바로 확인할 수 있도록 개선
- **수정 파일**: `js/app.js`, `js/storage.js`, `docs/db_schema.md`

#### 66. 운영 DB 기준으로 스키마 문서 길이 정보 정정
- **배경**: 설계 문서의 일부 컬럼 길이 정보가 실제 운영 DB와 달라 확인 기준으로 쓰기 어려웠음
- **변경 사항**:
  - 운영 DB probe 검증 결과 기준으로 `title`, `menu_name`, `screen_name`, `env_info`, `creator`, `assignee` 길이를 `VARCHAR(255)`로 정정
  - `screen_url`은 `TEXT`, `steps_to_repro`는 `TEXT`, `defect_identification`은 `VARCHAR(50)` 기준을 문서에 반영
  - 한글/영문 설계 문서와 Oracle 마이그레이션 가이드, 로컬 스키마 파일의 관련 값을 함께 정리
- **수정 파일**: `docs/db_schema.md`, `docs/program_design.md`, `docs/program_design_en.md`, `docs/java_spring_oracle_migration_guide.md`, `database/schema.sql`

#### 65. 외부 연동 URL 정규화 및 길이 초과 진단 보강
- **배경**: 운영 로그에서 `value too long for type character varying(255)` 오류가 반복됐고, 실제 원인 필드를 빠르게 특정할 수 있는 정보가 부족했음
- **변경 사항**:
  - 앱 저장 전 `screen_url`을 정규화해 불필요한 해시(`#...`)를 제거하고 일관된 URL 형식으로 저장하도록 보강
  - 외부 위젯과 Chrome 확장프로그램에서도 `screen_url`을 동일한 규칙으로 정규화하도록 맞춤
  - 저장 오류 로그 요약에 `screen_url_length`, `screen_url_origin`, `screen_url_query_length`를 추가
  - `varchar(n)` 길이 초과 오류 발생 시 의심 필드와 실제 길이를 함께 기록하도록 진단 정보를 강화
- **수정 파일**: `js/app.js`, `js/storage.js`, `docs/defectflow_report_widget_latest.js`, `extension/defectflow-reporter/background.js`

#### 64. 외부 연동 등록 실패 분석용 저장 오류 로그 추가
- **배경**: 테스트 대상 사이트에서 standalone 신규 결함 등록 시 저장 실패가 발생해도 원인 정보를 나중에 추적하기 어려웠음
- **변경 사항**:
  - 결함 저장 실패 시 중앙 DB 테이블 `defect_save_error_logs`에 구조화된 오류 로그를 저장하도록 확장
  - 중앙 저장이 실패해도 브라우저 localStorage에 최근 오류 로그를 로컬 백업으로 유지하도록 보강
  - 관리자 화면에 `저장 오류 로그` 탭을 추가해 중앙 로그 조회, 상세 내용 확인, JSON 다운로드, 로컬 백업 초기화가 가능하도록 구성
  - 외부 연동 캡처 이미지처럼 큰 inline 이미지가 들어오는 경우 저장 전에 한 번 더 최적화하도록 보강
  - Supabase 저장 실패 시 `stage`, `message`, `code`, `details`, `hint`를 구조화해 앱에서 활용할 수 있도록 정리
- **수정 파일**: `js/app.js`, `js/storage.js`, `docs/db_schema.md`, `database/create_defect_save_error_logs.sql`

#### 62. PC 신규 결함 등록의 모바일 전용 화면 버튼을 관리자 전용으로 제한
- **배경**: 일반 사용자에게는 모바일 전용 등록 버튼의 목적이 불명확해 혼선을 줄 수 있었음
- **변경 사항**:
  - PC 신규 결함 등록 화면의 `모바일 전용 화면 열기` 버튼을 관리자에게만 노출하도록 변경
  - 일반 테스터/조치자 화면에서는 버튼이 보이지 않도록 정리
- **수정 파일**: `js/app.js`

#### 61. Standalone 신규 결함 등록 초기 진입에서 대시보드 선렌더 제거
- **배경**: Chrome 확장프로그램 및 외부 연동으로 `?mode=standalone#register`에 진입할 때 대시보드가 먼저 렌더링된 뒤 등록 모달이 열리면서 화면 깜박임이 발생했음
- **변경 사항**:
  - 초기 `fetchData()` 완료 직후 대시보드를 먼저 렌더링하지 않고 초기 해시 라우트를 우선 처리하도록 순서 조정
  - standalone `register` 진입 시 대시보드 대신 전용 준비 셸을 렌더링한 뒤 등록 모달을 즉시 열도록 조정
  - 결과적으로 standalone 등록 진입에서 대시보드 선출력과 불필요한 첫 화면 전환을 제거
- **수정 파일**: `js/app.js`

#### 60. 대시보드 심각도 표를 심각도별 조치 현황으로 확대
- **배경**: 기존 `심각도별 결함 현황` 표는 심각도 건수만 보여줘 실제 조치 상태까지 한 번에 파악하기 어려웠음
- **변경 사항**:
  - 대시보드 표 제목을 `심각도별 조치 현황`으로 변경
  - 행 기준을 `심각도`, 열 기준을 `상태(Status)`로 재구성
  - 각 심각도별 `조치완료율`을 함께 표시
  - 전체 건수 및 상태별 건수를 클릭하면 해당 조건으로 결함 목록 필터 이동 가능하도록 연결
- **수정 파일**: `js/app.js`

#### 59. Chrome 확장프로그램 설치 안내 화면 및 다운로드 링크 추가
- **배경**: 신규 결함 등록 화면에서 확장프로그램 설치 경로와 다운로드 위치를 바로 안내할 필요가 있었음
- **변경 사항**:
  - 앱에 `Chrome 확장프로그램 설치 안내` 전용 화면을 추가
  - 설치 방법, 사용 방법, 주의사항, `extension.zip` 다운로드 버튼을 한 화면에서 제공
  - 신규 결함 등록 화면 하단에 `설치하기` 링크를 추가해 안내 화면으로 바로 이동할 수 있도록 연결
- **수정 파일**: `js/app.js`

## [2026-03-29] Standalone 등록 화면 로그인 강제

  - 모바일 결함 등록 버튼에서 이모지를 제거
  - 모바일 결함 등록 화면에 `메뉴명`, `화면명` 입력창을 추가
  - `결함 발생 화면` 선택 시 메뉴명과 화면명이 자동 입력되도록 연결

#### 57. 모바일 공통 위젯 실행 버튼 아이콘을 PC와 동일한 fa-bug로 통일
- **배경**: 모바일웹 `신고하기` 버튼은 이모지 아이콘을 사용하고 있어 PC 버전의 `Font Awesome fa-bug`와 형태 통일성이 맞지 않았음

  - 모바일 공통 위젯 실행 버튼 아이콘을 `🐞` 이모지에서 `Font Awesome 6.4.0`의 실제 `fa-bug`로 교체

#### 52. 신규 결함 등록 초기 진입 시 로그인 검증 강화
- **배경**: Chrome 확장프로그램과 `standalone/mobile` 등록 화면에서 저장된 브라우저 세션 정보만으로 로그인 여부가 판단되거나, 등록 모달 자체가 인증 없이 열릴 수 있는 여지가 있었음
- **변경 사항**:

  - 신규 결함 등록 모달 진입과 저장 시점에 모두 로그인 검증을 추가
  - 로그인 화면에 `등록 화면으로 자동 복귀` 안내를 표시해 extension 캡처 흐름을 유지

#### 53. 모바일 등록 문구 정리 및 신규 등록 기본 선택값 보정
- **배경**: 모바일 등록 화면의 문구를 더 간결하게 맞출 필요가 있었고, 신규 결함 등록 시 외부 연동 데이터가 비어 있으면 `심각도`와 `우선순위` 기본 선택값이 흔들릴 수 있었음
- **변경 사항**:
  - 모바일 간편 등록 화면 제목을 `모바일 결함 등록`으로 변경
  - `결함 발생 화면 (필수)` 라벨을 `결함 발생 화면`으로 변경
  - 신규 결함 등록 화면에서 `심각도=Minor`, `우선순위=Medium`이 항상 기본 선택되도록 보정
- **수정 파일**: `js/app.js`

#### 54. 세션 보유 시 등록 화면 로그인 깜빡임 방지
- **배경**: 신규 결함 등록 화면 실행 시 저장된 사용자 세션이 있어도 `DEFECTFLOW_DATA` 메시지가 세션 복구보다 먼저 도착하면 로그인 화면이 잠깐 먼저 보일 수 있었음
- **변경 사항**:
  - 세션 복구가 완료되기 전에는 외부 등록 데이터 수신 시 로그인 화면으로 즉시 이동하지 않도록 조정
  - 세션이 유효한 경우 로그인 화면을 거치지 않고 바로 등록 화면으로 진입하도록 보정
- **수정 파일**: `js/app.js`

#### 55. 캡처 엔진을 html2canvas-pro로 교체
- **배경**: 기존 `html2canvas 1.4.1` 기반 캡처는 최신 CSS 대응 범위가 제한적이어서, 운영 페이지 특성상 필요한 캡처 정확도를 더 안정적으로 확보할 필요가 있었음
- **변경 사항**:

  - 전역 노출 방식 차이를 흡수하는 로더를 추가해 기존 호출 구조를 유지
  - `same-origin` 자산 URL 치환, `useCORS: true`, `allowTaint: false` 등 기존 CORS 회피 보정 로직은 그대로 유지
- **수정 파일**: `docs/defectflow_report_widget_latest.js`, `test_view/test_view.html`
