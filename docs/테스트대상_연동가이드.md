# DefectFlow Integration Guide (Client-Side Widget)

이 가이드는 테스트 대상인 웹사이트에 **'결함 등록 버튼(Widget)'**을 추가하여, 테스트 중에 즉시 결함을 보고할 수 있도록 설정하는 방법을 설명합니다. 특히 최근 강화된 **보안 로그인** 및 **실시간 화면 캡처** 연동 방식을 포함하고 있습니다.

*   **관리 도구 URL**: [https://mohenz.github.io/defect_manage/](https://mohenz.github.io/defect_manage/)
*   **연동 예시 페이지**: [https://mohenz.github.io/defect_manage/test_view/test_view.html](https://mohenz.github.io/defect_manage/test_view/test_view.html)

## 1. 최신 연동 스크립트 (Recommended)

현재 권장 방식은 `html2canvas`를 사용하되, 캡처 직전에 상품 이미지와 일부 자산 URL을 **현재 페이지와 같은 origin으로 임시 치환**한 뒤 캡처하는 구조입니다. 테스트 대상 웹사이트의 `</body>` 태그 바로 위에 삽입하세요.

교체용 파일 경로:
- [defectflow_report_widget_latest.js](D:\Workspace\defect_manage\docs\defectflow_report_widget_latest.js)

실제 운영 반영 시에는 기존 구버전 위젯 스크립트를 제거하고, 아래 연결 코드만 남기는 방식을 권장합니다.

```html
<!-- DefectFlow Report Widget -->
<div id="defectflow-widget-container"></div>
<script src="https://mohenz.github.io/defect_manage/docs/defectflow_report_widget_latest.js"></script>
```

### 모바일웹 간편 등록 위젯

모바일웹에서는 캡처 없이 최소 정보만 빠르게 등록하는 전용 위젯을 사용할 수 있습니다. 테스트 대상 페이지의 `</body>` 바로 위에 아래 스크립트를 넣으면 우측 하단에 `신고하기` 버튼이 추가됩니다.

```html
<script>
window.DEFECTFLOW_MOBILE_REPORTER_CONFIG = {
  defaultScreenPath: 'FO (MO) > 메인화면 > 슬라이딩배너, 중분류',
  defaultTestType: '통합테스트',
  defaultSeverity: 'Minor',
  defaultPriority: 'Medium',
  defaultTitle: '[모바일웹] 결함 등록'
};
</script>
<script src="https://mohenz.github.io/defect_manage/docs/defectflow_mobile_report_widget.js"></script>
```

옵션 설명:
- `defaultScreenPath`: 모바일 `FO (MO)` 공통코드 기본값
- `defaultTestType`: 기본 테스트 구분
- `defaultSeverity`: 기본 심각도
- `defaultPriority`: 기본 우선순위
- `defaultTitle`: 기본 결함 제목

이 위젯은:
- 캡처를 수행하지 않습니다
- 현재 페이지 URL과 문서 제목을 함께 저장합니다
- `?mode=mobile#register` 화면으로 연결합니다

## 2. 주요 연동 포인트 및 변경 사항

### 2.1 URL 구조 (중요)
관리 시스템이 팝업('Standalone')임을 인식하고 사이드바를 숨기기 위해서는 파라미터 순서가 매우 중요합니다.
*   **Correct**: `index.html?mode=standalone#register`
*   **Incorrect**: `index.html#register?mode=standalone` (동작하지 않음)

### 2.2 보안 및 로그인 흐름
*   위젯을 통해 창이 열릴 때 **로그인이 되어 있지 않으면 자동으로 로그인 화면**으로 이동합니다.
*   로그인 완료 후에는 다른 페이지로 튕기지 않고, **원래 요청했던 결함 등록 화면으로 자동 복구**되어 캡처 데이터가 유지됩니다.

### 2.3 데이터 연동 (LocalStorage & postMessage)
*   **동일 도메인**: 위젯은 현재 사이트의 `localStorage`에 `pending_defect` 키로 데이터를 저장하여 전달합니다.
*   **도메인이 다른 경우 (Cross-Domain)**: 브라우저 보안 정책상 `localStorage` 공유가 불가능하므로, **`postMessage` 통신**을 사용합니다.
    *   신규 창(팝업)이 로드되면 부모 창에게 준비 완료(`DEFECTFLOW_READY`) 신호를 보냅니다.
    *   부모 창은 이를 확인하고 즉시 캡처 데이터(`DEFECTFLOW_DATA`)를 팝업으로 전송합니다.
    *   이 방식은 도메인이 달라도 안전하고 확실하게 데이터를 전달할 수 있는 표준 기술입니다.

### 2.4 캡처 품질 설정
*   `html2canvas`로 **현재 보이는 화면(뷰포트)** 을 원클릭 캡처합니다.
*   캡처 전에 `cos.emarteveryday.co.kr`, `cos-a.emarteveryday.co.kr` 계열 상품 이미지와 일부 자산 URL을 현재 페이지 origin으로 임시 치환해 CORS 영향을 줄입니다.
*   캡처 후에는 원래 자산 URL로 즉시 복원합니다.
*   캡처된 프레임은 `image/webp` 우선, 필요 시 `image/jpeg`로 자동 전환해 용량을 안정화합니다.
*   별도 공유 화면 선택 창 없이 동작합니다.

## 3. 작동 단계
1.  **클릭**: 위젯 버튼을 누르면 현재 화면에 보이는 영역을 즉시 캡처합니다.
2.  **치환**: 상품 이미지와 일부 자산 URL을 같은 origin 기준으로 임시 치환해 캡처 품질을 보정합니다.
3.  **전송**: 페이지 URL, 제목, User Agent와 함께 결함 등록 팝업으로 데이터가 전달됩니다.
4.  **완료**: 로그인 상태 확인 후, 모든 정보가 입력된 등록 폼이 나타납니다.
