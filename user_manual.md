# DefectFlow 사용자 매뉴얼 (User Manual)

## 1. 로그인 및 계정 보안
1.  **시스템 접속**: 제공된 URL을 통해 접속하면 로그인 화면이 나타납니다.
2.  **로그인**: 등록된 이메일과 비밀번호를 입력합니다.
3.  **비밀번호 초기화**: 최초 로그인 시 또는 관리자가 계정을 초기화한 경우, 비밀번호를 즉시 변경해야 시스템 이용이 가능합니다.
4.  **로그인 유지**: 브라우저를 닫아도 세션이 유지되며, 다시 접속 시 마지막으로 작업하던 화면으로 자동 연결됩니다.

## 2. 결함 등록 방법 (두 가지 방식)

### 2.1 내부 대시보드에서 등록
1.  **화면 이동**: 사이드바에서 **'결함 등록'** 메뉴를 선택합니다.
2.  **정보 입력**: 제목(5자 이상), 심각도, 우선순위, 재현 단계 등을 충실히 작성합니다.
3.  **이미지 첨부**: 로컬 PC의 이미지를 직접 업로드하거나 미리보기 영역을 통해 확인할 수 있습니다.

### 2.2 테스트 벤치(위젯) 연동 등록
1.  **위젯 실행**: 연동된 외부 사이트에서 '결함 리포트' 버튼을 클릭합니다.
2.  **자동 화면 캡처**: 버튼 클릭 시점의 현재 화면이 자동으로 캡처되어 리포트에 첨부됩니다.
3.  **자동 완성**: 페이지 URL, 화면명, 브라우저 정보가 자동으로 수집되므로 제목과 재현 단계만 입력하여 빠르게 등록 가능합니다.

## 3. 결함 조회 및 데이터 분석
1.  **대시보드**: 로그인 후 첫 화면에서 결함 통계 차트와 최근 등록된 20건의 결함을 실시간으로 확인합니다.
2.  **고급 검색**: '결함 목록' 메뉴에서 다음 조건으로 정밀 조회가 가능합니다.
    *   **심각도/상태**: 다중 선택을 통한 상태별 필터링
    *   **검색어**: 등록자나 담당자 성함으로 검색
    *   **기간**: 등록 일자 범위를 지정하여 검색
3.  **데이터 추출**: 목록 우측 상단의 **'엑셀 다운로드'** 버튼을 누르면 현재 필터링된 데이터가 CSV 파일로 저장됩니다.

## 4. 결함 조정 및 사후 관리
1.  **상세 조회**: 목록에서 결함을 클릭하여 상세 내용과 첨부된 스크린샷을 확인합니다.
2.  **조치 기록**: 담당자는 결함 처리 시 '조치 모달'을 열어 원인 분석, 조치 내용, 조치 기간을 입력하고 상태를 업데이트합니다.
3.  **상태 변경 권한**: 자신의 역할(테스터, 조치자, 관리자)에 따라 변경 가능한 필드와 메뉴가 제한될 수 있습니다.

## 5. 환경 설정 (관리자 전용)
*   관리자 계정으로 접속 시 사이드바 하단에 **'설정'** 메뉴가 노출됩니다.
*   시스템에서 사용할 '테스트 구분' 단계를 활성화하거나 비활성화하여 리포트 항목을 최적화할 수 있습니다.
