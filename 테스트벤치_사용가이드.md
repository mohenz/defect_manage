## 🧪 2. 테스트 환경 접속 방법
1.  **DefectFlow**에 로그인하여 메인 대시보드에 접속합니다.
2.  왼쪽 사이드바 메뉴 하단의 황금색 아이콘 **[🧪 테스트 벤치]**를 클릭합니다.
3.  새 탭으로 **ZeroStore Simulation** 화면이 열리며 즉시 테스트를 시작할 수 있습니다.

## 🎯 3. 집중 테스트 시나리오 (의도된 결함)
테스트 벤치에는 QA 역량을 테스트하기 위해 **3가지 유형의 의도된 결함**이 숨겨져 있습니다. 아래 시나리오를 따라 결함을 찾아보세요.

### 시나리오 A: UI 레이아웃 깨짐 (🎨 UI/UX Defect)
*   **발견 대상**: `프리미엄 도시락 세트`
*   **현상**: 상품 설명 텍스트가 너무 길어 상품 카드의 경계를 뚫고 나옵니다.
*   **테스트 포인트**: 텍스트 생략(Ellipsis) 처리 여부 및 모바일 뷰에서의 레이아웃 유지를 점검하십시오.

### 시나리오 B: 기능 작동 불가 (⚙️ Function Defect)
*   **발견 대상**: `초정밀 냉동고`
*   **현상**: 구매 버튼이 투명하게 변하거나 클릭 이벤트가 발생하지 않습니다.
*   **테스트 포인트**: 버튼의 `pointer-events` 속성 및 비활성화(Disabled) 상태의 시각적 피드백을 확인하십시오.

### 시나리오 C: 데이터 정합성/로직 오류 (💰 Logic/Data Defect)
*   **발견 대상**: `노이즈 캔슬링 헤드셋`
*   **현상**: 원래 가격 350,000원이 취소선 처리되고, 판매가가 **0원**으로 표시됩니다.
*   **테스트 포인트**: 비즈니스 로직 및 할인 계산 모듈의 데이터 정합성을 점검하십시오.

## 📝 4. 결함 보고 워크플로우 (Reporting)
버그를 발견했다면 다음 절차에 따라 시스템에 등록합니다.

1.  **버그 선택**: 테스트 화면 우측 하단의 **플로팅 벌레 버튼(FAB)**을 클릭합니다.
2.  **등록 폼 작성**: 자동으로 열리는 DefectFlow 등록 폼에 내용을 기입합니다.
    *   **등록자 정보**: 로그인한 사용자의 정보가 자동으로 입력됩니다.
    *   **캡처 데이터**: 발견 당시의 화면이 자동으로 연동되므로 상세 경로만 추가 기입하면 됩니다.
3.  **등록 완료**: [결함 등록] 버튼을 눌러 목록에 저장합니다.

---

## 🛠️ 5. 조치 및 상태 관리 (Action)
1.  DefectFlow의 **[결함 목록]**에서 방금 등록한 항목을 확인합니다.
2.  **조치자 계정**(`jisung@example.com` 등)으로 전환하여 로그인해 봅니다.
3.  목록 우측의 **[🔧 조치]** 버튼을 눌러 개발자 관점의 원인 분석 및 조치 내용을 입력하고 상태를 `Resolved`로 변경하십시오.

---

**💡 Tip**: 테스트 벤치 상단의 파란색 알림바에 안내된 주의사항을 읽어보시면 더욱 원활한 테스트가 가능합니다. 이제 실제 환경처럼 로그인하여 결함을 정복해 보세요!
